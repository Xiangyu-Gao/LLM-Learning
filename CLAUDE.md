First rule:

Please use first-principles thinking. You should not always assume that I clearly know what I want or how to achieve it. Remain thoughtful and cautious. Start from the fundamental needs and problems. If the motivation or goal is unclear, pause and discuss it with me. If the goal is clear but the path is not the most efficient, tell me and suggest a better approach.

Second rule:

The purpose of this repo is to learn through hands-on practice. When working on the weekly projects, please remember to create a study guide that helps me understand the code and learn from it. Also, please create a set of the most commonly asked interview Q&A pairs related to the topic.
